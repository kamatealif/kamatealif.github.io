PRJ.Book Recommendation Engine

Scraping, preprocessing, and similarity search Content-based recommendations from raw catalog data

This project is an end-to-end book recommendation system built to turn raw catalog data into a usable recommendation experience. The main idea is simple: if a user likes one book, the system should help them quickly discover other books with similar content and themes.

The pipeline starts with collecting book information from the source website using a scraper. That data includes titles, authors, descriptions, genres, and other metadata. After collection, the dataset is cleaned and organized so the search and recommendation steps can work on structured information instead of noisy raw text.

To generate recommendations, I use TF-IDF to turn book descriptions and related text fields into numerical representations. Once the books are represented in vector form, similarity search can retrieve titles that are contextually close to a selected book. This makes the project content-based, which is useful when working directly with book metadata rather than user behavior data.

The final step is presenting the results through a clean SvelteKit interface so recommendations are easy to browse instead of staying stuck inside a notebook or backend script. The project is not just about the model itself, but about building the full flow from data collection to user-facing exploration.

What the project is doing

  • Scraping book metadata from the source website.
  • Cleaning and organizing the dataset for reliable downstream use.
  • Converting text features into vectors with TF-IDF.
  • Finding similar books through content-based similarity search.
  • Presenting recommendations in a frontend that is easy to explore.

Why this approach

I chose a content-based recommendation approach because it works well even when user interaction history is limited. With strong metadata and description text, the system can still provide useful recommendations by comparing what books are about instead of depending on large user-rating datasets.

This project helped me understand how recommendation systems are built from real-world data problems: scraping, cleaning, feature construction, vectorization, retrieval, and then presenting the result in a way that actually feels useful.
