PRJ.Book Recommendation Engine

Scraping, preprocessing, and similarity search Content-based recommendations from raw catalog data

Problem

Book discovery becomes frustrating when a catalog is large but the browsing experience is weak. I wanted a system that could take one book and surface other relevant titles based on actual content instead of random popularity.

Why I built it

I wanted to understand recommendation systems end to end, not only as a model inside a notebook. That meant collecting the raw data myself, shaping it into a usable dataset, building the retrieval logic, and then presenting the output in a frontend that feels like a real product.

How it works

  • Scrape titles, authors, descriptions, and other metadata.
  • Clean the dataset so the important text fields are usable.
  • Transform descriptions into vectors with TF-IDF.
  • Retrieve similar books through content-based similarity search.
  • Show the recommendations through a SvelteKit interface.

Key decisions

  • Used content-based retrieval because rich metadata was available.
  • Chose TF-IDF as a strong baseline for text similarity.
  • Kept the frontend lightweight so the recommendation flow stays clear.
  • Designed the project as a full pipeline, not only a model demo.

What I learned

This project helped me understand recommendation work as a data systems problem: scraping, cleaning, feature construction, vectorization, retrieval, and presentation all matter. A useful recommender depends as much on clean data and sensible product flow as it does on the model itself.

What I'd improve next

  • Refine feature weighting across description, genre, and author fields.
  • Add better filtering and comparison inside the frontend.
  • Test stronger retrieval methods on top of the current baseline.
  • Introduce evaluation notes to compare recommendation quality over time.

Tech Stack

Python Scrapy Pandas Scikit-Learn TF-IDF SvelteKit Content-Based Retrieval

Project Snapshot

  • Data source: scraped public book catalog
  • Retrieval model: TF-IDF similarity search
  • Main goal: faster book discovery from content
  • User layer: SvelteKit browsing interface

Why it matters

This project shows how I think about recommendation systems as complete products, from raw data collection to the final user interaction layer.
