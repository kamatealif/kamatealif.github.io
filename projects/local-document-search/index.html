PRJ.Local Document Search Engine

Local-first search and reading workflow Built for deep offline study and slower thinking

Problem

When documents are spread across folders, notes, and reference files, even simple searching becomes slow. I wanted a local tool that could find the right document quickly and help me jump straight into reading.

Why I built it

The main reason for building this project was to support offline study sessions. I wanted to step away from the rush of the internet, slow down, think properly, and solve problems by working through my own notes and documents first.

That offline workflow helps me understand concepts more deeply and implement them with better reasoning instead of depending on constant external lookup.

How it works

  • Scan local folders and register documents into an index.
  • Extract readable text and useful metadata from the files.
  • Store structured entries for quick lookup and ranking.
  • Return relevant results with enough context to start reading fast.
  • Keep the full search and reading workflow on the local machine.

Key decisions

  • Kept the system local-first for privacy, speed, and focus.
  • Prioritized readable results over a complex search interface.
  • Designed it as a study tool, not only as a retrieval demo.
  • Focused on making result-to-reading flow feel immediate.

What I learned

This project taught me how much value there is in simple retrieval systems when they are built around a real workflow. Search quality is important, but so is the surrounding experience: what gets indexed, how results are ranked, and how quickly the user can move from query to understanding.

What I'd improve next

  • Add richer ranking signals beyond straightforward keyword matching.
  • Support better previews for larger technical documents.
  • Introduce stronger filtering by folder, file type, and tags.
  • Make incremental re-indexing more efficient as files change.

Tech Stack

Python FastAPI SQLite Search Indexing Text Extraction Local Retrieval

Project Snapshot

  • Main workflow: local indexing and ranked retrieval
  • Use case: offline study and focused reading
  • Data source: personal and local document files
  • Primary goal: faster search-to-reading flow

Why it matters

It turns scattered local files into something closer to a personal knowledge base, but without losing the quiet, offline-first workflow that motivated the project.
